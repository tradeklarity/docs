Events group multiple related markets together (e.g., "2028 US Election" contains multiple candidate markets).

## Response Format

```json
{

    {
      "event_id": "12345",
      "slug": "us-presidential-election-2028",
      "title": "US Presidential Election 2028",
      "description": "Who will win the 2028 presidential election?",
      "category": "politics",
      "subcategory": "elections",
      "image": "https://...",
      "active": true,
      "closed": false,
      "end_date": "2028-11-05T00:00:00Z",
      "creation_date": "2026-01-10T00:00:00Z",
      "age_seconds": 86400,
      "available_windows": ["5m", "15m", "30m", "1h", "4h", "24h"],

      "metrics": {
        "volume": 245000000,
        "liquidity": 12500000,
        "market_count": 15,

        "holders_no": 3300,

        "volume_24h": { "abs": 250000, "pct": 0.1, "status": "ok" },

```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `event_id` / `id` | string | Event identifier |
| `slug` | string | URL-friendly event slug |
| `title` | string | Event title |
| `description` | string | Event description |
| `category` / `subcategory` | string | Event classification |
| `image` / `icon` | string | Event imagery URLs |
| `active` / `closed` | boolean | Event status |
| `end_date` / `creation_date` | string | Event dates (ISO-8601) |
| `age_seconds` | number | Event age in seconds |
| `available_windows` | string[] | Delta windows with enough history |
| `tags` | string[] | Event tags (lowercase slugs) |
| `market_count` | number | Number of child markets |
| `markets` | array | Nested child markets with prices and metadata |

### Metrics Object

| Field | Type | Description |
|-------|------|-------------|
| `volume` | number | Total volume across all markets (USD) |
| `liquidity` | number | Total liquidity across all markets (USD) |
| `market_count` | number | Number of active markets in this event |
| `holders_total` / `holders_yes` / `holders_no` | number | Aggregate holder counts |
| `holders_whale` / `holders_smart_money` / `holders_insider` | number | Classification counts |

### Deltas Object

Same as markets: `{metric}_{window}` with `abs`, `pct`, `status`.

### Price Movement Metric

`price_movement` is event-specific — it aggregates price changes across all child markets:
- **`delta_abs`** — sum of absolute YES price deltas across all markets (total cents of movement)
- **`delta_pct`** — average of absolute YES price delta percentages (average % change per market)

<Note>
  `price_movement` does not support `since`-based deltas (only fixed windows).
</Note>

## Request Parameters

### Deltas

```json
{
  "deltas": [
    {"metric": "volume", "window": "24h", "compare": "delta_pct"},
    {"metric": "price_movement", "window": "1h", "compare": "delta_abs"}
  ]
}
```

**Metrics:** `volume`, `liquidity`, `market_count`, `holders_total`, `holders_yes`, `holders_no`, `holders_whale`, `holders_smart_money`, `holders_insider`, `price_movement`

**Windows:** `5m`, `15m`, `30m`, `1h`, `4h`, `24h`, `7d`, `30d`

**Custom `since`:** Same as markets — ISO-8601 timestamp, max 5, max 30 days. Not supported for `price_movement`.

### Sort, Filters, Pagination

Same interface as market discovery. See [Discover Markets](/api-reference/markets/post-discover) for detailed parameter documentation.

### Other Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `categories` | string[] | Filter by event category |
| `tags_any` / `tags_all` / `tags_none` | string[] | Tag filters |
| `search` | string | Text search on event title |
| `min_age_seconds` | number | Minimum event age |
| `require` | array | Only return events with valid deltas for specified metrics |
| `nulls` | string | Handle null deltas: `exclude`, `last` (default), `zero` |

## Example Requests

### Top Events by Volume

```bash
curl -X POST 'https://api.klarity.trade/events/discover' \
  -H 'Content-Type: application/json' \
  -d '{"sort": {"metric": "volume", "direction": "desc"}, "pagination": {"limit": 10}}'
```

### Trending Events (1h volume surge)

```bash
curl -X POST 'https://api.klarity.trade/events/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "deltas": [{"metric": "volume", "window": "1h", "compare": "delta_pct"}],
    "sort": {"metric": "volume", "window": "1h", "compare": "delta_pct", "direction": "desc"},
    "require": [{"metric": "volume", "window": "1h", "compare": "delta_pct"}],
    "pagination": {"limit": 20}
  }'
```

### Price Movers (1h)

```bash
curl -X POST 'https://api.klarity.trade/events/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "deltas": [{"metric": "price_movement", "window": "1h", "compare": "delta_pct"}],
    "sort": {"metric": "price_movement", "window": "1h", "compare": "delta_pct", "direction": "desc"},
    "pagination": {"limit": 20}
  }'
```

### Crypto Events

```bash
curl -X POST 'https://api.klarity.trade/events/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "tags_any": ["crypto", "bitcoin", "ethereum"],
    "sort": {"metric": "volume", "direction": "desc"},
    "pagination": {"limit": 20}
  }'
```

## API Constraints

| Constraint | Limit |
|-----------|-------|
| Filter nesting depth | 5 levels |
| Total filter conditions | 50 |
| Array fields | 50 items |
| String fields | 200 characters |
| Max deltas | 10 per request |
| Max since-deltas | 5 per request |
| Query timeout | 10 seconds |
| Pagination limit | 1-100 (default 20) |

## Error Responses

**400 — Validation Error**
```json
{"error": "Validation failed", "details": [{"path": ["deltas", 0], "message": "Exactly one of window or since must be provided"}]}
```

**504 — Query Timeout**
```json
{"error": "Query timeout — try simplifying your filters or reducing result size", "traceId": "..."}
```
