## Overview

Debug endpoint showing the current status of the insider tracker service, including connected clients and ClickHouse persistence statistics.

## Response

<ResponseField name="connectedClients" type="number">
  Number of SSE clients currently connected to the insider tracker stream
</ResponseField>

<ResponseField name="clickhouseEnabled" type="boolean">
  Whether ClickHouse persistence is enabled for trade storage
</ResponseField>

<ResponseField name="tradeCount" type="number | null">
  Total number of insider trades stored in ClickHouse. Null if ClickHouse is not configured.
</ResponseField>

<ResponseField name="oldestTimestamp" type="string | null">
  ISO 8601 timestamp of the oldest trade in ClickHouse. Null if no trades or ClickHouse not
  configured.
</ResponseField>

<ResponseField name="newestTimestamp" type="string | null">
  ISO 8601 timestamp of the newest trade in ClickHouse. Null if no trades or ClickHouse not
  configured.
</ResponseField>

<RequestExample>
```bash cURL
curl "https://ws.klarity.trade/stream/insider-tracker/stats"
```

```javascript JavaScript
const response = await fetch('https://ws.klarity.trade/stream/insider-tracker/stats');
const stats = await response.json();
console.log('Connected clients:', stats.connectedClients);
console.log('ClickHouse enabled:', stats.clickhouseEnabled);
console.log('Trade count:', stats.tradeCount);
```

</RequestExample>

<ResponseExample>

## Storage Details

Insider trades are persisted to ClickHouse for backfill support:

- Trades passing floor thresholds are stored (notional &gt;= $1,000, unique markets &lt;= 20, price 0.15-0.85)
- 30-day TTL - older trades are automatically cleaned up
- Backfill queries filter by user's config from the stored trades
