---
title: Market Trade Stream
openapi: openapi-stream.yaml GET /markets/{slug}/stream
description: SSE stream of enriched trades for a specific market
---

Establishes a Server-Sent Events (SSE) connection that streams real-time enriched trades for a specific market. Each trade includes wallet classification flags (whale, smart money, insider) and account-level stats.

For historical trades, use [`GET /markets/{slug}/trades`](/api-reference/markets/get-trades) before connecting.

## Path Parameters

<ParamField path="slug" type="string" required>
  Market slug identifier
</ParamField>

## Response

Server-Sent Events (SSE) stream. Each event contains an enriched trade.

### Event Types

| Event       | Description            |
| ----------- | ---------------------- |
| `init`      | Connection established |
| `keepalive` | Heartbeat (every 15s)  |
| `trade`     | New enriched trade     |

</ResponseField>
<ResponseField name="isSmartMoney" type="boolean">

</ResponseField>
<ResponseField name="isInsider" type="boolean">
  Whether the trader is classified as an insider

</ResponseField>

</ResponseField>

<RequestExample>
```javascript JavaScript
const eventSource = new EventSource(
  "https://ws.klarity.trade/markets/will-trump-win-2024/stream"
);

eventSource.addEventListener("init", (event) => {
  console.log("Connected:", JSON.parse(event.data));
});

eventSource.addEventListener("trade", (event) => {
  const trade = JSON.parse(event.data);
  if (trade.isSmartMoney || trade.isPortfolioWhale) {
    console.log("Whale/Smart money trade:", trade);
  }
});

eventSource.onerror = () => {
  console.log("Connection lost, reconnecting...");
};
```

```bash cURL
curl -N "https://ws.klarity.trade/markets/will-trump-win-2024/stream"
```
</RequestExample>

<Note>
The stream automatically reconnects if the connection drops. Use `EventSource` in browsers for automatic reconnection handling. Note that `EventSource` requires listening on the `trade` event name (not `message`).
</Note>

## Backfill on Reconnect

When a client reconnects after a disconnect, it should:

1. **Track the last trade timestamp** received before disconnect
2. **Fetch missed trades** via `GET /markets/{slug}/trades` filtered by that timestamp
3. **Reconnect to stream** and deduplicate any overlap

This ensures no trades are missed during network interruptions.
