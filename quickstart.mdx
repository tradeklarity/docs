## Subscribe to a realtime stream

The Telegram and X monitors are plain WebSocket endpoints. Connect, and you
start receiving every enriched message as it arrives.

<CodeGroup>
```js Node.js
import WebSocket from "ws";

const ws = new WebSocket("wss://telegram.klarity.trade");

ws.on("message", (raw) => {
  const msg = JSON.parse(raw.toString());
  if (msg.type === "connected") {
    console.log("connected:", msg.clientId);
    return;
  }
  if (msg.type === "message") {
    console.log(`[${msg.channelTitle}] ${msg.text}`);
  }
});
```

```js Browser
const ws = new WebSocket("wss://x.klarity.trade");

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);
  if (msg.type === "message") {
    console.log(`@${msg.authorUsername}: ${msg.text}`);
  }
};
```
</CodeGroup>

The first frame is always a `connected` envelope; every following `message`
frame is a broadcast. See [Telegram](/streams/telegram) and
[X (Twitter)](/streams/twitter) for the full payload shapes.

## Call the UMA API

The UMA API is REST over HTTPS, versioned under `/v1`. The health check needs
no authentication:

```bash
curl https://uma.klarity.trade/v1/health
# {"status":"ok","timestamp":"2026-06-11T12:24:58.154Z"}
```

Data endpoints require a Klarity session cookie. Once signed in to the
dashboard, your browser sends the `klarity_access_token` cookie automatically:

```bash
curl https://uma.klarity.trade/v1/markets?active=true \
  --cookie "klarity_access_token=<your-session-token>"
```

See the [UMA API overview](/uma/overview) for authentication details and the
full endpoint reference.
