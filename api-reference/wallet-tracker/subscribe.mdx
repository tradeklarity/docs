---
title: Subscribe to Wallet
openapi: openapi-stream.yaml POST /wallets/subscribe
description: Subscribe to real-time trade notifications for a wallet address
---

## Overview

Subscribe to receive real-time trade notifications for a Polymarket wallet address. When the wallet executes a trade, you'll receive an event via the SSE stream.

Optionally provide metadata (color, icon, alias) that will be stored with the wallet subscription. If the wallet is already tracked, these fields will be updated (upsert behavior).

**Limits:**

- Maximum 50 wallets per user
- Subscriptions are idempotent (subscribing again updates metadata if provided)

## Authentication

Send your JWT as either:
- **Cookie:** `klarity_access_token=<jwt>` (automatic for same-origin browser requests)
- **Header:** `klarity_access_token: <jwt>` (for server-side requests from Next.js API routes)

The user ID is extracted from the JWT `sub` claim.

## Request Body

<ParamField body="wallet" type="string" required>
  Ethereum wallet address to track. Must be a valid 42-character hex address starting with `0x`.

Example: `0x5aa981521b4e94fee3b3bf18fe441f62e865febe`

</ParamField>

<ParamField body="color" type="string">
  Hex color code for wallet visualization (e.g., `#FF5733`). Must match pattern `^#[0-9a-fA-F]{6}$`.
  Optional.
</ParamField>

<ParamField body="icon" type="string">
  Icon identifier (alphanumeric, underscores, hyphens; max 50 characters). Must match pattern
  `^[a-zA-Z0-9_-]+$`. Optional.
</ParamField>

<ParamField body="alias" type="string">
  Custom wallet nickname or alias (max 100 characters). Optional.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the subscription was created
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable status message
</ResponseField>

<ResponseField name="wallet" type="string">
  Normalized (lowercased) wallet address
</ResponseField>

<RequestExample>
```bash cURL
# Using header
curl -X POST "https://ws.klarity.trade/stream/wallets/subscribe" \
  -H "Content-Type: application/json" \
  -H "klarity_access_token: eyJhbGciOiJIUzI1NiJ9..." \
  -d '{"wallet": "0x5aa981521b4e94fee3b3bf18fe441f62e865febe"}'

# With metadata (color, icon, alias)
curl -X POST "https://ws.klarity.trade/stream/wallets/subscribe" \
  -H "Content-Type: application/json" \
  -H "klarity_access_token: eyJhbGciOiJIUzI1NiJ9..." \
  -d '{
    "wallet": "0x5aa981521b4e94fee3b3bf18fe441f62e865febe",
    "color": "#FF5733",
    "icon": "whale",
    "alias": "Big Whale Wallet"
  }'
```
</RequestExample>

<ResponseExample>
```json
{
  "success": true,
  "message": "Subscribed to wallet",
  "wallet": "0x5aa981521b4e94fee3b3bf18fe441f62e865febe"
}
```
</ResponseExample>

## Error Responses

**401 - Missing or invalid JWT:**

```json
{
  "error": "Missing authentication token",
  "code": "UNAUTHORIZED"
}
```

**400 - Invalid wallet address:**

```json
{
  "error": "Invalid request",
  "details": [{ "path": ["wallet"], "message": "Invalid Ethereum address" }]
}
```

**400 - Wallet limit reached:**

```json
{
  "success": false,
  "message": "Maximum 50 wallets per user"
}
```
