---
title: Wallet Tracker Stream
openapi: openapi-stream.yaml GET /wallets/stream
description: SSE stream of trades for user-subscribed wallets (ticket auth)
---

Requires a single-use ticket from `POST /wallets/stream-ticket`. On connect, recent trades are sent as individual `event: trade` messages (backfill), then live trades follow.

## Query Parameters

<ParamField query="ticket" type="string" required>
  Single-use JWT ticket from `POST /wallets/stream-ticket` (30s TTL).
</ParamField>

## SSE Events

### `event: system`

```json
{ "type": "connected", "wallets": ["0x742d35cc6634c0532925a3b844bc9e7595f42cbe"] }
```

Also emits `subscribed`, `unsubscribed`, and `error` messages.

<RequestExample>
```bash cURL
# 1. Get ticket
TICKET=$(curl -s -X POST https://ws.klarity.trade/stream/wallets/stream-ticket \
  -H "klarity_access_token: YOUR_JWT" | jq -r '.ticket')

# 2. Connect
curl -N "https://ws.klarity.trade/stream/wallets/stream?ticket=$TICKET"
```

```typescript TypeScript
const es = new EventSource(`https://ws.klarity.trade/stream/wallets/stream?ticket=${ticket}`);

es.addEventListener('trade', (e) => {
  const trade: TradeAlertEvent = JSON.parse(e.data);
});

es.addEventListener('system', (e) => {
  const msg = JSON.parse(e.data);
});
```
</RequestExample>
