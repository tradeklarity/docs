---
title: 'Category Performance'
openapi: 'GET /users/{address}/category-performance'
description: 'Get per-category performance breakdown for a trader'
---

## Overview

Returns P&L, volume, position count, and win rate broken down by market category (tag). Useful for understanding a trader's strengths and weaknesses across different market types (politics, sports, crypto, etc.).

**Use this endpoint for:**

- Trader profiling by market category
- Identifying category-specific edge or weakness
- Building category-level performance dashboards

Data is sourced from the `wallet_tag_stats` table, populated by the `rollup-wallet-advanced-stats` sync job. If no data is available for a trader, the response returns an empty `categories` array with `updatedAt: null`.

## Path Parameters

<ParamField path="address" type="string" required>
  Wallet address (0x format)
</ParamField>

## Response

<ResponseField name="address" type="string">
  User's wallet address (lowercase)
</ResponseField>

<ResponseField name="categories" type="array">
  <Expandable title="Category performance item">
    <ResponseField name="tag" type="string">
      Market category tag (e.g. "Politics", "Sports", "Crypto")
    </ResponseField>
    <ResponseField name="pnl" type="number">
      Realized PnL in USD for this category
    </ResponseField>
    <ResponseField name="volume" type="number">
      Total trading volume in USD for this category
    </ResponseField>
    <ResponseField name="positionCount" type="number">
      Number of closed positions in this category
    </ResponseField>
    <ResponseField name="winRate" type="number">
      Win rate for this category (0-1, e.g. 0.72 = 72%)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="updatedAt" type="string | null">
  ISO timestamp of the most recent data update, or `null` if no data exists for the trader
</ResponseField>
