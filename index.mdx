Klarity ingests market-moving signals and Polymarket resolution data, then exposes them through two kinds of interfaces:

<CardGroup cols={2}>
  <Card title="Realtime streams" icon="bolt" href="/streams/overview">
    WebSocket feeds of Telegram and X (Twitter) messages, each enriched with AI
    translation, importance scoring, and related Polymarket events.
  </Card>
  <Card title="UMA API" icon="scale-balanced" href="/uma/overview">
    A REST API over Polymarket markets and their UMA optimistic-oracle
    resolutions — proposals, disputes, and DVM votes.
  </Card>
</CardGroup>

## Endpoints at a glance

| Service | Type | URL |
| --- | --- | --- |
| Telegram monitor | WebSocket | `wss://telegram.klarity.trade` |
| X (Twitter) monitor | WebSocket | `wss://x.klarity.trade` |
| UMA API | REST | `https://uma.klarity.trade` |

## How it fits together

The monitors watch their sources continuously and broadcast every message to
connected clients in real time. The UMA pipeline watches Polymarket market
resolutions on-chain (Polygon mempool, with a subgraph backfill) and on
Ethereum (DVM votes), and serves the aggregated history through the UMA API.

<Note>
  The realtime streams are open WebSocket endpoints. The UMA API requires a
  Klarity session cookie — see [Authentication](/uma/overview#authentication).
</Note>
