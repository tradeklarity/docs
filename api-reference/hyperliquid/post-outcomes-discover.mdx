---
title: 'Discover HIP-4 Outcomes'
openapi: openapi-hyperliquid.yaml POST /hyperliquid/outcomes/discover
description: Cursor-paginated HIP-4 outcome discovery with deltas and lifecycle stages
---

Cursor-paginated **outcome** discovery for Hyperliquid HIP-4. Outcomes are the tradeable binaries (≈ Polymarket markets); they can exist standalone or grouped in questions (≈ Polymarket events).

Returns full upstream HIP-4 metadata:
- `questionId` (null for standalone outcomes — HL-specific feature)
- `description` (raw pipe-encoded, e.g., `"class:priceBinary|underlying:BTC|targetPrice:..."`)
- Parsed fields: `class`, `underlying`, `targetPrice`, `quoteToken`
- Per-window deltas (volume/trades/mid prices/holder counts)
- Lifecycle stage classification

**Use cases:**
- Drill deep into outcome-level metrics and signals
- Use `POST /hyperliquid/questions/discover` for a unified question+standalone feed with embedded outcomes

Backed by `hl_outcome_discover_serving` — a single-phase query against a
materialized view refreshed every 5 minutes.

Rate limit: 30 requests/minute.

## Lifecycle Stages

| Stage       | Definition |
|-------------|-----------|
| `new`       | First seen within the last 24h |
| `building`  | Positive volume momentum |
| `peak`      | Stable, high-volume |
| `resolving` | Close to expiry |
| `quiet`     | Low activity, no recent change |

## Delta Metrics & Windows

**Metrics:** `volume`, `trades`, `yes_mid_px`, `no_mid_px`, `holders_total`,
`holders_yes`, `holders_no`, `holders_whale`

**Windows:** `5m`, `15m`, `30m`, `1h`, `4h`, `24h`, `7d`, `30d`

**Compare:** `delta_abs` (absolute change) or `delta_pct` (percent change)

Response delta keys are emitted as `{metric}_{window}` (e.g. `volume_1h`).

## Sort Fields

`volume_24h`, `trades_24h`, `holders_total`, `holders_yes`, `holders_no`,
`holders_whale`, `holders_smart_money`, `yes_mid_px`, `no_mid_px`, `expiry_at`,
`first_seen_at`

Pass `window` + `compare` on the sort spec to sort by a delta value instead of
a static column.

## Examples

### Hottest movers (1h volume surge)

```bash
curl -X POST 'https://api.klarity.trade/hyperliquid/outcomes/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "deltas": [{"metric": "volume", "window": "1h", "compare": "delta_pct"}],
    "sort": {"metric": "volume_24h", "direction": "desc"},
    "pagination": {"limit": 20}
  }'
```

### New & rising

```bash
curl -X POST 'https://api.klarity.trade/hyperliquid/outcomes/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "stages": ["new", "building"],
    "sort": {"metric": "volume_24h", "direction": "desc"},
    "pagination": {"limit": 20}
  }'
```

### Whale radar

```bash
curl -X POST 'https://api.klarity.trade/hyperliquid/outcomes/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "deltas": [{"metric": "holders_whale", "window": "24h", "compare": "delta_abs"}],
    "sort": {"metric": "holders_whale", "direction": "desc"},
    "pagination": {"limit": 20}
  }'
```

### Filter by underlying + holders range

```bash
curl -X POST 'https://api.klarity.trade/hyperliquid/outcomes/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "underlying": "BTC",
    "holdersMin": 50,
    "volumeMin": 10000,
    "sort": {"metric": "first_seen_at", "direction": "desc"},
    "pagination": {"limit": 20}
  }'
```

### Threshold conditions

```bash
curl -X POST 'https://api.klarity.trade/hyperliquid/outcomes/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "conditions": [
      {"metric": "volume_24h", "op": "gt", "value": 50000},
      {"metric": "holders_total", "op": "gte", "value": 100}
    ],
    "conditionLogic": "and",
    "pagination": {"limit": 20}
  }'
```

### Cursor pagination

```bash
curl -X POST 'https://api.klarity.trade/hyperliquid/outcomes/discover' \
  -H 'Content-Type: application/json' \
  -d '{
    "pagination": {"limit": 20, "cursor": "eyJpZCI6IjE2NSIsInZhbHVlIjoxMjAwMH0="}
  }'
```
